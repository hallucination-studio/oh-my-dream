# Quality Score

## Priority Areas

First priority: canvas workflow and canvas state management. Second priority: project management and navigation around the canvas. Third priority: all remaining supporting features and integrations.

## Scoring Dimensions

- Product correctness
- UX and operator clarity
- Architecture and maintainability
- Reliability and observability
- Security and data handling

## Usage

- Score changes by affected domain and layer, then weight the final judgment toward the highest-priority area touched by the work.
- A change that weakens canvas clarity, persistence, or recoverability should not pass on the strength of polish elsewhere.
- Document recurring weak spots and improvement themes here.
