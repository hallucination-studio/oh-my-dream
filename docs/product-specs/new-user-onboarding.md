# New User Onboarding

## Outcome

The first successful experience is: a user opens the client, understands that projects and configuration live locally, creates or opens a project, enters the canvas, and completes a small generation or planning loop without losing context.

## First-Session Requirements

- The product should explain the local-first model clearly enough that users understand their project state stays on their machine.
- The path from home page to project to canvas should feel continuous and low-friction.
- Provider configuration should be understandable and reversible, especially when entering API keys or choosing generation backends.
- A first-session user should be able to recover from a mistake or interruption without feeling that the project disappeared.
